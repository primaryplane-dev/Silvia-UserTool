# コンベア一覧表 改修メモ（2026-07-06）

## 対応背景
- ユーザー要望:
  - コンベアチェック一覧表を開いたとき、生産終了分が「作業終了清掃時・中間確認」に表示されてしまう。
  - 例: R31に出ているデータは AP31 列に表示する。

## 原因
- 実績展開で `BGFNCD='1'`（生産終了）を通常明細と同じ時刻ブロック計算に流していたため、
  生産終了分がR列など途中ブロックに表示されることがあった。

## 修正内容
- 対象: src_コンベア一覧表/Bas_List.bas
- `subEditList` の実績ループ内で以下を実施:
  - `isProductionEnd = (CStr(RS("BGFNCD") & "") = "1")` を追加。
  - 生産終了分は時刻ブロック計算を通さず、`writeCol = 42`（AP列）へ固定。
  - 通常データは従来どおり `F -> L -> R -> X -> AD -> AJ -> AP` の時刻ブロックで表示。

## 影響範囲
- 明細の表示列制御のみ。
- 行特定ロジック（`fncFindRow` / BC列キー）や件数判定SQLは変更なし。
- 「製造終了」ヘッダ表示（`ST.Cells(1, CL_SY + 1)`）の判定は従来どおり。

## 動作確認結果
- ユーザー確認:
  1. 生産終了分がAP列に表示される。
  2. 作業終了清掃時・中間確認側へ同データが出ない。
  3. 製造終了表示が従来どおり出る。

## 保守メモ
- 今後、`BGFNCD='1'` の列配置要件を変更する場合は、
  `subEditList` の `isProductionEnd` 分岐のみを調整し、通常時刻ブロック処理には影響させないこと。
